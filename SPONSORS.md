# Sponsor MorsVitaEst

If MVE is useful to you, sponsorship keeps it independent and shipping.

Built solo, open source, no VC funding. The project's architecture (user-owned data, vendor-agnostic, local-first) is structurally hostile to the kinds of monetization other AI apps default to — subscription tiers tied to features, locking users into a single vendor, mining usage data for training. Sponsorship is the route that doesn't compromise the thesis.

## Tiers

| Tier | Amount | What you get |
|---|---|---|
| **Supporter** | $5 / month | Your name in this file. Knowledge that you're keeping the lights on. |
| **Backer** | $25 / month | Above + name in a "Backed by" section in the app's About screen. Early access to preview builds before they go to the rolling channel. |
| **Hero** | $100 / month | Above + a private channel to influence the roadmap. Direct line on bugs and feature requests. Logo / link on the project landing page. |
| **Underwriter** | $500 / month | Above + dedicated support hours each month. Custom-built MCP servers / skills / integrations on request. Co-marketing if you want it. |

## Where the money goes

- **Compute** — CI minutes, ARM cross-compile, hosted MCP servers for testing.
- **Devices** — physical Android hardware for testing across versions / manufacturers.
- **Services** — Apple Developer / Play Store fees once we get there. F-Droid submission costs nothing but takes time.
- **Time** — fewer client hours = more MVE hours. The single largest lever.

## Other ways to help

- **Star the repo** — it's free and lifts the project in discovery.
- **File issues** — bug reports and feature requests sharpen the roadmap.
- **Share it** — if MVE solves a problem you have, someone else probably has the same problem.
- **Code contributions** — PRs welcome. The architecture (`app is the source of truth, AI is the messenger`) is the load-bearing constraint; everything else is iterable.
